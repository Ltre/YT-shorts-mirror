# 长辈数据画像开发计划

## 背景

管理后台本次先上线 cookies 录入、服务器缓存管理、浏览记录查看。长辈数据画像涉及隐私解释、指标稳定性和展示方式，先拆成计划，不在本轮直接开发。

## 数据来源

- `profiles.json` 中的 `events`、`watchHistory`、`liked`、`disliked`、`favorites`、`searches`。
- `videos.json` 中的视频标题、作者、标签、时长、来源、缓存状态。
- 后续可追加设备侧信息：访问时间段、网络状态、播放失败、重复观看次数。

## 一期画像指标

1. 兴趣标签：按观看时长、点赞、收藏、跳过、点踩综合打分。
2. 作者偏好：识别常看作者与被快速跳过作者。
3. 内容节奏：偏好短视频/中视频、是否喜欢循环重看。
4. 活跃时间：按小时统计观看与互动峰值。
5. 负反馈主题：点踩、快速跳过高频标签，作为降权依据。

## 后台展示

- 总览卡片：观看数、喜欢数、收藏数、近 7 日活跃天数。
- 标签雷达/条形榜：展示 top 标签和负反馈标签。
- 时间热力：按星期和小时展示观看峰值。
- 明细解释：每个画像结论都能点开看到最近触发事件，避免“黑盒推荐”。

## 推荐系统接入

- 画像只输出可解释权重，不直接覆盖原始事件。
- 推荐排序继续由 `recommender.js` 计算，但增加画像快照作为输入。
- 每次画像计算生成版本号和时间戳，便于回滚与对比。

## 隐私与安全

- 默认只保存在本机 `data` 目录。
- 后台增加“清空画像/导出画像”按钮前，需要二次确认。
- 不展示 cookies 原文，只展示文件状态；cookies 修改保留最近更新时间。

## 开发步骤

1. 新增 `server/profile-insights.js`，从 profile + videos 生成画像快照。
2. 新增 `data/profile-insights.json` 缓存计算结果。
3. 管理后台新增“数据画像”页签，只读展示画像。
4. 加入手动重算接口 `/api/admin/profile-insights/rebuild`。
5. 再把画像摘要接入 `/api/feed` 的 `profileSummary`。
